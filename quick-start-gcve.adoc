= 使用 VMware 工作负载迁移顾问迁移到 Google Cloud VMware Engine 的快速入门
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
开始使用 VMware 迁移顾问，将您当前的基础架构迁移到使用 Google Cloud NetApp Volumes 作为外部 NFS 数据存储的 Google Cloud VMware Engine。

在开始之前，您应该了解 https://docs.netapp.com/us-en/workload-setup-admin/permissions-reference.html["NetApp Workload Factory 的权限"^]。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png["一个"]登录工作负载工厂
[role="quick-margin-para"]
你需要 https://docs.netapp.com/us-en/workload-setup-admin/sign-up-saas.html["在 Workload Factory 上创建一个帐户"^]并使用以下任一方式登录 https://docs.netapp.com/us-en/workload-setup-admin/console-experiences.html["控制台体验"^]。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-2.png["两个"]添加凭据和权限
[role="quick-margin-para"]
选择 https://docs.netapp.com/us-en/workload-setup-admin/permissions-reference.html["权限策略"]满足您的需求。

[role="quick-margin-para"]
如果选择不授予权限，则可以开始使用 Workload Factory for VMware 复制部分完成的代码示例。

[role="quick-margin-para"]
如果选择授予权限，则需要 https://docs.netapp.com/us-en/workload-setup-admin/add-credentials.html["手动向帐户添加凭据"]提供具有必要权限的 GCP 服务帐户，以查看和管理 Google Cloud NetApp Volumes 资源、VPC 配置和 Google Cloud VMware Engine 数据存储区连接。

[role="quick-margin-para"]
https://docs.netapp.com/us-en/workload-setup-admin/add-credentials.html["了解如何添加凭据和权限"^](英文)

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-3.png["三个"]上传虚拟机清单数据
[role="quick-margin-para"]
您可以使用规划中心上传有关当前 VM 环境的库存详细信息。使用数据收集器脚本、RVTools 或NetApp Data Infrastructure Insights捕获当前的 VM 配置并将数据上传到规划中心。

[role="quick-margin-para"]
link:upload-vm-inventory.html["了解如何上传虚拟机清单数据"^] 。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-4.png["四个"] 使用VMware迁移顾问创建部署计划
[role="quick-margin-para"]
启动 VMware 迁移顾问，并使用 Google Cloud NetApp Volumes 作为外部 NFS 数据存储配置要在 Google Cloud VMware Engine 基础架构上创建的虚拟机环境。

[role="quick-margin-para"]
link:launch-migration-advisor-gcve.html["了解如何使用VMware迁移顾问创建部署计划"]。
